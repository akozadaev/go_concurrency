## 1. Горутины и планировщик

**Горутина** - легковесная нить, старт с `go`. Планировщик Go мультиплексирует горутины на потоки ОС (**M:N**); блокировка одной горутины (канал, I/O, `Sleep`) не обязана удерживать поток.

**Конкурентность ≠ параллелизм:** порядок вывода без синхронизации **не детерминирован**.

Вход в курс: [`lessons/01-goroutines`](lessons/01-goroutines) - `main` завершается раньше дочерних задач, вывод может пропасть.

## 2. Каналы и `select`

| Вид | Поведение |
|-----|-----------|
| **Небуферизованный** `make(chan T)` | Рендеву отправки и приёма. |
| **Буферизованный** `make(chan T, n)` | До `n` отправок без блокировки, пока буфер не полон. |

**`close`:** после закрытия `for range ch` завершается; отправка в закрытый канал - паника.

**`select`:** из готовых веток выбирается **одна псевдослучайно** - порядок `case` в коде не гарантирует приоритет.

Примеры: [`lessons/02-channels-wait`](lessons/02-channels-wait), [`lessons/03-channels-range`](lessons/03-channels-range).

## 3. Пакет `sync`

| Примитив | Назначение |
|----------|------------|
| **`Mutex` / `RWMutex`** | Взаимное исключение. |
| **`WaitGroup`** | Ожидание группы горутин. |
| **`Once`** | Однократное выполнение. |
| **`Cond`** | Ожидание условия под мьютексом. |
| **`Map`** | Потокобезопасная карта для особых паттернов доступа. |
| **`Pool`** | Пул временных объектов (не кэш с гарантией хранения). |

Уроки **`05`–`13`** в [`lessons/`](lessons/) проходят по этим темам от data race к `Pool`.

**Гонки:** `go run -race` / `go test -race`.
