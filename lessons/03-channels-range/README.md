# Канал: `close` и `for range` (урок 03)

**Запуск:** `cd lessons/03-channels-range && go run .`  
Задания: [../EXERCISES.md](../EXERCISES.md).

## Что делает код

- **`producer`** в горутине отправляет в канал значения `10, 20, …, 50` и вызывает **`close(ch)`**, чтобы сообщить потребителю: новых значений не будет.
- **`main`** читает канал циклом **`for val := range ch`**. Цикл завершается, когда канал **закрыт** и все уже отправленные элементы прочитаны.

Канал **небуферизованный** (`make(chan int)`): каждая отправка синхронизируется с приёмом - продюсер и потребитель идут в ногу по одному значению.

## Ожидаемый вывод

```
 10 20 30 40 50
```

Перед первым числом есть пробел из-за `fmt.Printf(" %d", val)`.

## Замечания

- Читать из **закрытого** канала можно: получите нулевое значение типа и `false` из двухзначной формы `v, ok := <-ch`. Цикл `range` по закрытому каналу просто заканчивается.
- **Писать** в закрытый канал нельзя - будет паника.
- Если убрать `close(ch)`, `for range` в `main` **зависнет** после пяти значений: потребитель будет ждать шестую отправку, которой не будет.
