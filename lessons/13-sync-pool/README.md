# `sync.Pool`

## Идея

**`sync.Pool`** - пул временных объектов: **`Get()`** достаёт значение (или вызывает **`New`**), **`Put()`** возвращает его в пул. Планировщик GC может **очистить** пул между циклами сборки - нельзя считать, что положенный объект обязательно дождётся следующего `Get`.

Подходит для **краткоживущих буферов** и аллокаций на горячем пути (HTTP-ответы, сериализация). Не заменяет **кэш** с гарантией хранения.

## Запуск

```bash
cd lessons/13-sync-pool
go run .
```

## Важно

- После **`Get`** привести тип к нужному (`(*bytes.Buffer)`).
- Перед использованием вызвать **`Reset()`** (или обнулить поля), если объект мог остаться грязным.
- Не держите ссылки на объект после **`Put`**.
